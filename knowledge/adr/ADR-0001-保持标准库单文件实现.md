# ADR-0001-保持标准库单文件实现

## 状态

已取代。后续以 [ADR-0002-拆分核心逻辑并增加调试和打包入口](ADR-0002-拆分核心逻辑并增加调试和打包入口.md) 为准。

## 背景

项目当前目标是提供一个可以直接运行的黑白翻转棋桌面脚本。功能范围较小，主要包括棋盘规则、Tkinter 界面和简单电脑对手。

如果过早引入包结构、第三方 GUI 框架或复杂 AI，会提高运行门槛和维护成本。

## 决策

历史上曾保持 `reversi.py` 单入口、标准库实现：

- GUI 使用 Tkinter。
- 依赖文件 `requirements.txt` 仅说明依赖 Python 标准库。
- 不引入第三方棋类引擎或 GUI 框架。
- 仓库通过 README、AGENTS、CLAUDE 和 knowledge 补足可维护性。

## 影响

当时的优点：

- 用户可以直接运行 `python .\reversi.py`。
- 无需安装第三方依赖。
- 适合小工具和教学演示。

后续暴露的限制：

- 单文件规模继续增长后可读性会下降。
- 简单 AI 不适合高强度对弈。
- 自动化测试尚未拆出独立规则模块，后续可考虑重构。

## 涉及改动

- `README.md`
- `AGENTS.md`
- `CLAUDE.md`

- `.gitignore`
- `knowledge/`

## 验证

```powershell
python -m py_compile .\reversi.py
```

该验证仅适用于本 ADR 被取代前的单文件阶段。当前验证命令见 ADR-0002 和 `knowledge/02_运行与交付/03_验证与交付.md`。

## 运行逻辑

历史默认运行路径为 `python .\reversi.py`。当前桌面入口已调整为 `python .\tk_run.py`，`reversi.py` 仅作为兼容入口保留。
