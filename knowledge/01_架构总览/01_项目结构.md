# 项目结构

## 当前结构

```text
.
├── core.py
├── tk_ui.py
├── tk_run.py
├── reversi.py
├── build_nuitka.ps1
├── build_android.ps1
├── sync_android_assets.ps1
├── requirements.txt
├── android/
├── pic/
│   ├── background.png
│   ├── qizi0.png
│   └── qizi1.png
└── knowledge/
```

## 文件职责

- `core.py`：核心规则、棋盘规格、AI 难度策略和联机模式骨架，不依赖 UI。
- `tk_ui.py`：Tkinter 桌面界面，负责资源加载、窗口缩放、棋盘渲染和用户交互。
- `tk_run.py`：Tkinter 启动入口，也是 Nuitka 打包入口。
- `reversi.py`：兼容入口，转发到 `tk_run.py`。
- `build_nuitka.ps1`：Nuitka 打包脚本，默认使用 envs2，输出 `VsingerReversi.exe`。
- `android/`：Kotlin + Jetpack Compose 原生 Android 工程，独立于 Python 运行时。
- `build_android.ps1`：Android 构建入口，先同步素材，再调用 Gradle 构建 APK。
- `sync_android_assets.ps1`：把根目录 `pic/` 素材复制到 Android `drawable-nodpi`。
- `pic/background.png`：棋盘背景。
- `pic/qizi0.png`：言和棋素材。
- `pic/qizi1.png`：天依棋素材。
- `requirements.txt`：Pillow 和 Nuitka 打包依赖。

## 模块边界

- `ReversiGame`：纯规则层，负责棋盘状态、合法落子、翻子、计分、胜负判断，支持 6x6、8x8、10x10、12x12。
- `choose_ai_move()`：AI 选择逻辑，支持简单、中等、高难。
- `NetworkSession`：联机对弈占位接口，后续可替换为真实网络实现。
- `ReversiApp`：Tkinter 界面层，负责窗口、棋盘绘制、鼠标事件、状态显示、电脑落子调度。
- `ReversiCore.kt`：Android 规则层，行为参考 `core.py`，但不直接运行 Python 代码。
- `MainActivity.kt`：Android Compose 界面层，负责移动端布局、棋盘触控、状态显示和电脑落子调度。

## 跨端边界

- Python 桌面版和 Android 版是双实现，不会自动同步代码改动。
- 修改 `tk_ui.py` 只影响 Tkinter 桌面界面。
- 修改 `core.py`、AI 策略、棋盘规格、模式语义或用户可见棋子名称时，必须检查 `android/app/src/main/java/com/vsinger/reversi/ReversiCore.kt`。
- 修改 `pic/` 后，执行 `sync_android_assets.ps1` 并重新构建 Android APK。
