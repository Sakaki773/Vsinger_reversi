## OneFile 单文件交付

桌面版正式交付需要使用 Nuitka onefile 模式：

```powershell
powershell -ExecutionPolicy Bypass -File .\build_nuitka.ps1 -LowMemory -OneFile
```

该模式输出单个 `VsingerReversi.exe`，适合发给最终用户直接运行。普通目录模式主要用于定位打包问题或检查依赖文件。

Nuitka 打包要求项目名称和项目路径使用英文/ASCII，不要放在包含中文的目录下。例如使用 `C:\Users\admin\Desktop\VsingerReversi`，不要使用 `C:\Users\admin\Desktop\黑白翻转棋`。`build_nuitka.ps1` 会在发现非 ASCII 路径时直接失败，避免进入编译阶段后才出现乱码或找不到文件。

未明确要求时，不主动执行 Nuitka 打包。

## Android 验证

首次命令行构建前，可执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\setup_android_tools.ps1
```

该脚本会在 `.android-tools/` 下准备本地 JDK、Gradle 和 Android SDK，不修改 Python `envs2`。

构建 Debug APK：

```powershell
powershell -ExecutionPolicy Bypass -File .\build_android.ps1
```

`build_android.ps1` 会优先使用 `.android-tools/` 中的本地工具链；若不存在，再尝试 `android/gradlew.bat` 或系统 `gradle`。`envs2` 是 Python 环境，不提供 Android 构建工具。

默认 APK 输出：

```text
android/app/build/outputs/apk/debug/app-debug.apk
```

Android 不使用 Nuitka onefile 模式；Android 的单文件交付物是 APK。需要发布时保留 APK 构建说明，不要把 Streamlit 或 Tkinter 入口作为 Android 打包入口。

修改 `pic/` 后，应先执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\sync_android_assets.ps1
```

当前已支持通过项目本地 `.android-tools/` 工具链完成 Debug APK 构建。发布签名包需要另行配置签名证书。
